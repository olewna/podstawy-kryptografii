Yarn

Od dzisiejszych zajęć, na każdych laboratoriach będziemy tworzyć projekt za pomocą menadżera paczek – Yarna. Yarn jest już zainstalowany w salach laboratoryjnych, więc jeżeli pracujesz na komputerze uczelnianym możesz śmiało pominąć pierwszy podpunkt.

  • Zainstaluj Yarna za pomocą komendy 
    npm install --global yarn
  • Sprawdź, czy Yarn jest zainstalowany i w jakiej wersji 
    yarn -v
  • Stwórz nowy folder dla dzisiejszych laboratoriów, a wewnątrz tego folderu wykonaj komendę 
    yarn init
    Wprowadź nazwę projektu (zwyczajowo powinna być taka sama, jak nazwa folderu, w którym ją wywołujemy), a resztę opcji zatwierdź klawiszem enter. W folderze pojawi się plik package.json oraz yarn.lock.

ESLint

Zaczniemy również korzystać z narzędzia o nazwie ESLint. Wewnątrz utworzonego folderu wykonaj polecenie

yarn add eslint --dev

Parametr --dev oznacza, że dodana paczka będzie służyła wyłącznie developmentowi i nie będzie instalowana na produkcji.

Od tej pory należy pilnować, aby tworzony do projektu folder node_modules nie trafił na repozytorium. W tym celu tworzymy plik .gitignore z nazwą wspomnianego folderu. Jeżeli chcesz dokładnej informacji, czego nie powinno wrzucać na repozytorium zajrzyj do gotowego, rozbudowanego pliku Node.gitignore.

Później, po pobraniu projektu z repozytorium (niezawierającego node_modules z modułami niezbędnymi do działania programu) możemy doinstalować wszystkie potrzebne paczki, jakie zostały określone w pliku package.json komendą yarn install.

Dlaczego folderu node_modules nie należy wrzucać na repo?

Następnie uruchom inicjalizację eslint poleceniem

yarn run eslint --init

W konsoli pojawi się parę opcji do ustawienia. Wybierz kolejno: 

How would you like to use ESLint? · problems
What type of modules does your project use? · commonjs
Which framework does your project use? · none
Does your project use TypeScript? · No
Where does your code run? · node
What format do you want your config file to be in? · JSON

Wszystkie powyższe opcje można zmieniać edytując plik .eslintrc.json.

Następnie utwórz folder src i wewnątrz niego utwórz skrypt index.js. Wklej do niego poniższy kod:

switch(status) {
  case 'Completed':
    runFunc();
 
  case 'Running': 
    runFunc2();
}

Powyższy kod wygeneruje błędy ESLint – korzystająć z podpowiedzi edytora, napraw je. Działanie eslinta możemy sprawdzić na kilka sposobów:

  1. Instalując rozszerzenie ESLint w VS Code
  2. Wywołując komendę eslint '**/*.js' opisaną w konfiguracji projektu (instrukcja niżej)

Nodemon

Kolejną paczką, którą dodamy jest Nodemon, który wykrywa zmiany w kodzie i automatycznie ponownie uruchamia skrypt. W głównym folderze projektu wykonaj polecenie:

yarn add --dev nodemon

Następnie możesz uruchomić konkretną paczkę za pomocą:

npx nodemon src/index.js

Polecenie npx uruchamia zainstalowane lokalnie paczki (w tym przypadku nodemon, analogiczne działanie dla eslinta). Zmień w dowolny sposób skrypt, a następnie zapisz go. Zauważ, że skrypt został ponownie uruchomiony, po zapisie.

Ostatnim krokiem będzie edycja pliku package.json (plik zawierający konfiguracje projektu). Dodaj poniższy kod w dowolnym miejscu podanego pliku (zachowując strukturę pliku json):

  1. Napisz funkcję (nie korzystając z mechanizmu async/await ani Promise'ów): 
    const poKolei = (fun1, fun2, cb) => { ... };
    taką, że:
    • jej dwoma pierwszymi argumentami są funkcje asynchroniczne – możesz założyć, że funkcje te muszą być przygotowane na przyjęcie określonej listy argumentów, z których korzystać będzie poKolei
    • funkcja poKolei powinna zapewnić, że fun2 wykona się zawsze po fun1, a wyniki wygenerowane przez fun1 zostaną przekazane do fun2.
    • trzecim argumentem jest „callback” cb, czyli funkcja, której zadaniem jest przetworzenie wyników zwracanych przez fun2
  2. Napisz funkcję (nie korzystając z mechanizmu async/await ani Promise'ów): 
    const razem = (fun1, fun2, cb) => { ... };
    taką, że:
    • jej dwoma pierwszymi argumentami są funkcje asynchroniczne – ponownie możesz założyć, że funkcje te muszą być przygotowane na przyjęcie określonej listy argumentów, z których korzystać będzie razem
    • funkcja razem powinna zapewnić, że fun1 oraz fun2 będą wykonywać się „równolegle”, a wyniki przez nie wygenerowane zostaną przekazane – jako tablica [wynik1, wynik2] do funkcji cb.
    • trzecim argumentem jest „callback” cb, czyli funkcja, której zadaniem jest przetworzenie wyników zwróconych przez fun1 i fun2.
  3. Napisz funkcję: 
    const grupuj = (funTab1, funTab2, cb) => { ... };
    spełaniającą poniższe warunki:
    • Dwoma pierwszymi argumentami funkcji grupuj są tablice funkcji asynchronicznych zwracające wartość liczbową.
    • Funkcja grupuj powinna zapewnić, że wszystkie funkcje z obu tablic będą wykonywać się „równolegle”.
    • Funkcja grupuj powinna wywoływać funkcję cb, która przyjmuje jako parametr pogrupowane wyniki z wszystkich funkcji w formacie: 
      [
  [ wynik_funkcji_1_z_funTab1, wynik_funkcji_1_z_funTab2 ],
  [ wynik_funkcji_2_z_funTab1, wynik_funkcji_2_z_funTab2 ],
  [ wynik_funkcji_3_z_funTab1, wynik_funkcji_3_z_funTab2 ],
  [ wynik_funkcji_4_z_funTab1, wynik_funkcji_4_z_funTab2 ],
  [ wynik_funkcji_5_z_funTab1, wynik_funkcji_5_z_funTab2 ],
  // ...
]
      i drukuje je w konsoli w dowolny sposób.
    • Jeżeli liczba funkcji w obu tablicach nie jest równa, to funkcja grupuj powinna uzupełniać brakujące wyniki wartością 0.
  4. Napisz funkcję: 
    const connect = (funTab, fun) => { ... };
    spełaniającą poniższe warunki:
    • Pierwszym argumentem funkcji connect jest tablica funkcji asynchronicznych, zwracających wartość liczbową.
    • Drugim argumentem connect jest funkcja asynchroniczna, której jedynym argumentem jest wartość liczbowa, a wynikiem również wartość liczbowa.
    • Funkcja connect powinna zapewnić, że wszystkie funkcje przekazane w parametrze funTab będą wykonywać się „równolegle”.
    • Funkcja connect powinna zwracać pogrupowane wyniki w formacie: 
      [
  [ wyn_1, fun(wyn_1) ],
  [ wyn_2, fun(wyn_2) ],
  [ wyn_3, fun(wyn_3) ],
  [ wyn_4, fun(wyn_4) ],
  [ wyn_5, fun(wyn_5) ],
  // ....
]
      gdzie wyn_i oznacza funTab[i]().

    razem(promisesTab, callback) {
  // ...
